01 — Pendahuluan

Apa itu Routing?

Routing adalah proses pengiriman paket data melalui jaringan dari satu perangkat ke perangkat lainnya. Dalam dunia jaringan komputer, terdapat dua metode utama untuk menentukan jalur data: Static Routing dan Dynamic Routing.

Static Routing

Konfigurasi manual oleh admin

Karakteristik

  • Admin memasukkan rute secara manual ke tabel routing.
  • Jalur tidak berubah otomatis jika ada kegagalan link.
  • Tidak ada pertukaran informasi antar router.

Kelebihan

  • Beban CPU sangat ringan
  • Kontrol penuh oleh admin
  • Tidak pakai bandwidth untuk update

Kekurangan

  • Tidak praktis untuk jaringan besar
  • Tidak ada failover otomatis
  • Risiko human error tinggi
⚡

Dynamic Routing

Otomatis via protokol

Karakteristik

  • Router saling bertukar informasi tabel routing secara otomatis.
  • Adaptif: mencari jalur alternatif jika ada link yang putus.
  • Sangat cocok untuk jaringan menengah hingga besar.

Protokol Routing Populer

RIP
Routing Information Protocol Penentu jalur: hop count (lompatan). Maks 15 hop. Cocok jaringan kecil.
OSPF
Open Shortest Path First Standar terbuka, sangat cepat. Menggunakan algoritma Dijkstra.
EIGRP
Enhanced IGRP (Cisco) Milik Cisco, gabungan keunggulan RIP dan OSPF.
BGP
Border Gateway Protocol Protokol utama antar ISP di Internet. Backbone jaringan global.

Kelebihan

  • Mudah dikelola di jaringan besar
  • Konvergensi cepat dan otomatis
  • Konfigurasi lebih sederhana

Kekurangan

  • Memakan resource CPU & RAM
  • Butuh bandwidth untuk update
  • Keamanan butuh enkripsi tambahan

05 — Panduan Praktis

Kapan Menggunakan Mana?

Pilih metode routing yang tepat sesuai kondisi jaringan kamu.

// Gunakan Static Routing jika:

  • Jaringan kecil (misalnya hanya 2–3 router)
  • Ingin kontrol penuh atas lalu lintas data
  • Router memiliki spesifikasi hardware yang sangat rendah
  • Jalur jaringan tidak sering berubah

// Gunakan Dynamic Routing jika:

  • Mengelola jaringan perusahaan besar (Enterprise)
  • Bekerja sebagai penyedia layanan internet (ISP)
  • Topologi jaringan sering berubah
  • Membutuhkan failover otomatis jika jalur putus

// Catatan: Teknik Hybrid

Dalam praktik di lapangan, seringkali digunakan teknik Hybrid — jaringan utama menggunakan Dynamic Routing, namun untuk akses ke internet atau server tertentu tetap menggunakan Static (Default Route). Ini memberikan fleksibilitas sekaligus kontrol yang optimal.
